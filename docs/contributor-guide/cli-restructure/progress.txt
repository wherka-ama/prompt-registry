Locked architectural decisions (see decisions.md):
- Single binary: `prompt-registry` (11 bin/*.js become subcommands).
- Framework (locked iter 13): primary = clipanion, fallback = oclif, eliminated = citty/commander/yargs.
- Config loader: c12 (UnJS) + YAML; cargo-style upward walk; defaults<user<project<env<--config FILE<--config KEY=VALUE<flags (8-step chain locked iter 18).
- Output flag: unified `-o, --output {text,json,yaml,markdown,table,ndjson}` with auto-detect (text on TTY, json off-TTY) and stable JSON envelope `{schemaVersion, command, status, data, warnings, errors, meta}` (locked iter 20).
- Exit codes: 3-tier scheme — POSIX (0–2), sysexits.h (64–78), domain (100+) (locked iter 21).
- Errors: `RegistryError {code, message, hint, docsUrl, cause, context}` with 11 dotted-path namespaces, ~60 codes (locked iter 23).
- Plugin model: PATH-binary discovery (`prompt-registry-<name>`, gh/kubectl model) primary; npm-plugins opt-in only via `plugins.allowNpm: true` with lockfile-pinning requirement (locked iter 27–28).
- Phase 2 architectural invariants: leaf commands never import clipanion directly; only `lib/src/cli/framework/` does. `Context` is the only IO surface; ESLint forbids direct `node:fs`/`node:net`/`process.env`/`process.exit`. (Locked iter 13 + iter 25.)
- Test tiers: unit / framework / command / golden / e2e. Coverage targets: framework 90%, command 70%, e2e narrow (install + one-per-noun). (Locked iter 25.)
- Migration: 6-line shims under `lib/bin/<old-name>.js` generated from a single `_shims.json` manifest. Deprecation lifecycle Phase 4→5→next-major; `--exit-on-deprecation` is the CI-strictness lever. (Locked iter 29.)
